Data Model

  • An instance of the CSBAppDataModel class, saved in the current instance of AppDelegate
  • Can be accessed from any point within the app by calling AppDelegate.dataModelInstance
  • Contains properties that save various portions of the data:
    • metadata : An instance of the CSBMetadataModel struct containing the various metadata properties of this survey
    • dataPoints : An array of CSBDataPoint structs, containing properties of a data entry
  • Can be saved to/retrieved from a permanent storage file called PersistentCopy.json using the save() and retrieve() methods
  • Contains bitmask flags to represent various data states important to how/if views load data
    • The verified flag should be set to true after data verification, false if any more changes are made, and checked before sending data

Data Entry 1 Screen

  • No objects should be displayed by this screen, as it should be unnavigatable at this point
  • Displayed as a Table View, it contains 1 field per cell (Except the last cell which also contains a reference selector)
  • The next button is initially disabled
  • Everytime a form is modified, a form validation method runs on the page to determine if the data was inputted properly
    • If the validation fails, the labelFailed(for:[String:String]) method is called with the forms that are incorrect and the reason
      • [ ] THIS METHOD MUST BE IMPLEMENTED BY THE UI TEAM TO FUNCTION
    • If the validation succeeds on all forms, the next button is enabled
    • If the validation fails and the next button was already enabled, it is disabled
  • When the next button is tapped, all entered data is saved to the AppDelegate.dataModelInstance.metadata struct, and the data model is saved to file
  • If the data model already contains metadata, loads data to view

Photo Selector

  • This view controller is presented by any of the data entry pages whenever a photo needs to be selected/taken
  • Any VC that uses the selector must conform to CSBUnwindableViewController (See documentation for requirements)
  • To instantiate the VC, call let vc = UIStoryboard(name: "PictureSelector", bundle: nil).instantiateViewController(withIdentifier: "PictureSelector")
  • If selecting multiple images, set the stack property to an array of the names of the images
    • To start at an image other than the first, also set the stackIndex property
  • If selecting a single image, set the imageTitle property to the name of the image
